# spec-check-review

验证 review 文档中的问题是否真实存在，补充遗漏，然后修复代码。

## 执行步骤

### 1. 读取 review 文档

从 $ARGUMENTS 获取 review 文档路径，读取全部内容。

### 2. 验证已有问题

逐一检查文档中列出的每个问题：

- 读取问题指向的代码文件和行号范围
- 确认问题是否真实存在
- 验证问题描述中的"期望行为"和"实际行为"是否准确
- 对每个问题标注验证结果：
  - `[确认]` — 问题存在，描述准确
  - [`误报`]` — 问题不存在或描述不准确，注明原因
  - `[需调整]` — 问题存在但描述需要修正，更新问题描述

### 3. 补充遗漏问题

在验证已有问题之后，基于同样的代码范围做一次独立审查：

- 检查是否有 review 文档遗漏的问题
- 参照 review 文档的问题分类（`[偏离]` / `[逻辑]` / `[缺陷]`）
- 补充的问题追加到文档末尾，标注严重级别
- 补充的问题初始状态为 `[todo]`

### 4. 修复代码

只修复 `[确认]` 和 `[需调整]` 的问题，以及补充的新问题：

- 按严重级别排序，CRITICAL 优先
- 每修复一个问题，更新文档中对应的"修复状态"：
  - `[todo]` → `[done]`
  - 跳过的更新为 `[skip]` 并注明原因
- `[误报]` 的问题标记为 `[skip]`，注明"验证为误报"

### 5. 报告

修复完成后，简要报告：
- 验证结果：确认几个、误报几个、调整几个
- 补充了几个新问题
- 修复了几个、跳过了几个
- 还有哪些未修复（如有）
